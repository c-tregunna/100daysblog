Let's talk variables and naming conventions. It's best to be descriptive with naming variables in JS. This, I guess, is common sense. However it could be so easy to name something name or job. But what name? What job?

Variables are SO important in JS, so it's vital to understand them, how to name them and how to use them. A few things to consider when naming a variable:

  • It's best to use camelCase when naming. Why, cause you should k.
  • There are reserved keywords which you can't use as these will throw and error. Here is a list
  • There are 3 ways to declare a variable, var, let and const. **
  • Ultimately, the variable is there to store a value

So think of a variable like a box. The variable name is the label on the box and the value you gave to the variable is the contents of the box.

What is camelCase? Is it luggage for a camel?

Errr...no. Simply put it is a way of writing something. So if you are using more than one word, the first word starts in lowercase, in the second word, the first letter is capitilized, in the third word, the first letter is capitilized and so on. See this in action on the variable below.
